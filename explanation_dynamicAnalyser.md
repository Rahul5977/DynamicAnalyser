# What is DynamicAnalyser? — A Simple Explanation

> **For anyone — no tech background needed!**

---

## The Problem It Solves (The "Why")

Imagine you work at a company and every time a developer writes new code and wants to "ship" it — that is, make it live for real users — their computer runs a long checklist of automated tasks to make sure nothing is broken. This checklist is called a **CI/CD pipeline** (think of it like a quality-control assembly line for software).

The problem is: **this checklist takes time.** Sometimes a lot of time. And when it gets slow, developers are stuck waiting. They don't know _which part_ of the checklist is the culprit, or _why_ it slowed down, or _what to do about it_.

> Imagine waiting 20 minutes every time you hit "save" on a document just to find out it didn't crash — that's frustrating, right?

**DynamicAnalyser** is a tool that watches this assembly line, figures out exactly what is slowing it down, and tells developers how to fix it — in plain language, with specific suggestions.

---

## What Does It Actually Do? (The "What")

Think of DynamicAnalyser as a **smart performance doctor for software pipelines**. Here's what it does, step by step:

### 1. 🔍 It Watches the Assembly Line
It connects to GitHub (the platform where developers store and share code) and reads the "logbooks" — detailed records of every step that ran during the assembly line, along with how long each step took.

### 2. 🗺️ It Reads the Source Code
It also reads the actual code files in the project — like reading the instruction manual — to understand what each step of the assembly line is *really doing* under the hood.

### 3. 🔗 It Connects the Dots
It links the slowness in the logbook back to the specific lines of code that caused it. Instead of saying "Step 5 was slow," it can say "Step 5 was slow because of *this specific function* in *this specific file*."

### 4. 🤖 It Uses AI to Diagnose the Root Cause
It then sends all this information to an AI assistant (Claude, made by Anthropic — similar to ChatGPT) and asks: "Why is this slow? What's the real underlying problem?" The AI analyzes everything and gives a clear explanation.

### 5. 💡 It Recommends Fixes
Not only does it explain the problem, it also ranks the best fixes — from most impactful to least — and estimates exactly how many seconds each fix would save. It even shows the developer the specific code change to make.

### 6. 📊 It Shows Everything in a Visual Dashboard
All of this analysis is shown in a clean, visual interface (a web app) with charts, color-coded timelines, and cards showing each recommendation — so it's easy to understand at a glance.

> **Think of it like a GPS navigation app — but for code performance.**
>
> - The GPS watches traffic (= DynamicAnalyser watches pipeline logs)
> - It identifies the bottleneck (= slow step on the assembly line)
> - It re-routes you (= it suggests code fixes)
> - It tells you how much time you'll save (= estimated saving in seconds)
>
> Without the GPS, you'd have to guess why you're stuck in traffic and manually try different routes. With it, you're guided precisely and efficiently.

## The Goal (The "Vision")

> **Cut the time developers wait by at least 25%** — for example, shrinking a 20-second wait to 15 seconds or less.

That might sound small, but developers run this pipeline hundreds of times a day across a whole team. Over a year, that adds up to **thousands of hours saved** — time that can be spent building better products instead of waiting.

The bigger vision is to make CI/CD pipelines *self-improving*: the tool learns what commonly causes slowdowns (it knows 7 recurring "bad habits" in code, called **anti-patterns**) and proactively catches them before they become problems.

## What Makes It Special?

Most tools only tell you **what** is slow (e.g., "the test step took 45 seconds"). DynamicAnalyser goes two steps further:

1. **It tells you WHY** — by linking the slow step back to the actual code causing it.
2. **It tells you HOW to fix it** — with ranked, ready-to-apply code suggestions generated by AI.

It also understands **five programming languages** (Python, JavaScript, TypeScript, Java, Go), so it works for almost any modern software project.

## The Key Features at a Glance

- **🔥 Flamegraph** — A colorful chart showing each pipeline step and how long it took. Green = fast, amber = medium, red = slow.
- **🤖 AI Root Cause Analysis** — Plain-language explanation of why a step is slow.
- **💬 Fix Suggestions** — Specific code changes, ranked by how much time they'd save.
- **📈 Trends Over Time** — Charts showing if your pipeline is getting faster or slower week by week.
- **🔔 Automatic Alerts** — Can be set up to automatically analyze a pipeline the moment it finishes running.
- **✅ Developer Feedback** — Developers can mark suggestions as "helpful" or "not helpful," which helps the system improve over time.
- **📄 Downloadable Reports** — Export the full analysis as a document to share with your team.

## A Day in the Life — How a Developer Uses It

1. **Sarah pushes new code** to GitHub and her pipeline starts running.
2. **DynamicAnalyser automatically picks it up** via a webhook (an automatic notification system).
3. The pipeline finishes in 22 seconds — *slower than usual*.
4. Sarah opens the DynamicAnalyser dashboard and sees a **flamegraph** with a big red bar on the "Install Dependencies" step.
5. She clicks "Run Analysis." The AI comes back and says:
   > *"Your 'npm install' step is downloading packages from scratch every single run because there's no caching configured. This is the root cause of the 8-second slowdown."*
6. The tool shows her the exact config change to add caching — a 3-line fix rated as **"low effort"** with an **estimated saving of 7,200 ms** (7.2 seconds).
7. Sarah applies the fix. Her pipeline now runs in 14 seconds. ✅
8. The dashboard records the improvement and shows the trend line going down.

## Quick Glossary (Plain-English Definitions)

| Term | What It Means |
|------|--------------|
| **CI/CD Pipeline** | An automated assembly line that tests and deploys code |
| **Log / Logbook** | A detailed record of everything that happened during a pipeline run |
| **Bottleneck** | The slowest step that holds everything else up |
| **AST (Abstract Syntax Tree)** | A way to read and understand code like a map |
| **LLM / AI** | A powerful AI (like ChatGPT or Claude) that understands and generates text |
| **Anti-pattern** | A known bad habit in code that causes problems |
| **Flamegraph** | A visual chart showing how long each step took, colored by speed |
| **Webhook** | An automatic notification sent between software systems |
| **Diff** | A highlighted comparison showing exactly what changed in code |
| **GitHub** | The most popular platform for storing and sharing code |
| **Caching** | Saving a result so you don't have to recalculate it next time |
| **Tree-sitter** | A tool that reads source code to understand its structure |
